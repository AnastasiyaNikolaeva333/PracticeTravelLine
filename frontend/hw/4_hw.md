## Домашнее задание 4: Обновление графика валют через `useEffect`

### Цель

Доработать конвертер из третьей домашки: добавить график изменения курса выбранной валютной пары и организовать его автоматическое обновление через `useEffect`.

Основной фокус задания - корректная работа с эффектами: зависимости, очистка таймеров/запросов, обработка ошибок и удаление лишних `useEffect`, если данные можно вычислить без них.

Под графиком в этом задании имеется в виду line chart с историей курса для текущей валютной пары. Нужно отобразить не одно текущее значение, а массив точек из `GET /prices`, где каждая точка показывает курс пары в конкретный момент времени.

### Что нужно реализовать

#### График курса

1. Добавить в интерфейс график истории курса для текущей валютной пары.
2. Данные для графика получать из backend через существующий эндпоинт:
   - `GET /prices?paymentCurrency=XXX&purchasedCurrency=YYY&fromDateTime=...&toDateTime=...`.
3. На графике должны отображаться точки из ответа API:
   - ось X - `dateTime`;
   - ось Y - `price`;
   - tooltip или подпись точки с датой/временем и значением курса приветствуются.
4. Иными словами, если API вернул массив из 10 элементов, на графике должно быть 10 точек, соединенных линией в хронологическом порядке.
5. Пользователь должен видеть, как курс менялся во времени, а не только последнее значение.
6. При изменении валютной пары график должен сразу перезагружаться для новой пары.
7. Последний элемент истории курса считать актуальным курсом:
   - обновлять отображаемый курс в шапке конвертера;
   - использовать его для пересчета результата конвертации;
   - обновлять дату/время актуального курса в UI.

#### Переключение периода

1. Добавить переключатель периода графика: `1 MIN`, `2 MIN`, `3 MIN`, `4 MIN`, `5 MIN`.
2. Активный период должен визуально отличаться от остальных.
3. При смене периода график должен перезагружаться без перезагрузки страницы.
4. Период в этом задании - это временное окно, за которое нужно показать историю.
5. Пример: если сейчас `12:00:00` и выбран период `3 MIN`, нужно запросить данные за промежуток от `11:57:00` до `12:00:00`.
6. `fromDateTime` вычислять от текущего времени назад на выбранное количество минут.
7. `toDateTime` - опциональное свойства.
8. Клиент сам вычисляет `fromDateTime`/`toDateTime` и запрашивает нужный диапазон.
9. Так как backend добавляет новые цены примерно каждые `10 секунд`, в периоде `1 MIN` обычно будет около 6 точек, в `5 MIN` - около 30 точек.

#### Автоматическое обновление через `useEffect`

1. Реализовать автоматическое обновление графика через `useEffect`.
2. График должен обновляться:
   - сразу при первом открытии страницы после загрузки валют;
   - при изменении валютной пары;
   - при изменении выбранного периода;
   - автоматически по таймеру без действий пользователя.
3. Интервал обновления выбрать осознанно. Рекомендуемое значение - `10 секунд`, так как backend добавляет новые цены примерно с такой частотой.
4. В `useEffect` обязательно сделать cleanup:
   - очищать `setInterval`/`setTimeout`;
   - отменять текущий запрос через `AbortController` или безопасно игнорировать устаревший ответ.
5. Нельзя допускать race condition: если пользователь быстро меняет валюты или период, старый ответ API не должен перезаписать данные нового графика.

#### Состояния загрузки и ошибок

1. Для графика хранить отдельные состояния:
   - `loading`;
   - `data`;
   - `error`.
2. Если график загружается впервые, показать loader или skeleton в области графика.
3. Если ошибка произошла при первой загрузке графика, показать понятное сообщение в области графика.
4. Если ошибка произошла при автообновлении, не очищать старый успешный график; показать неблокирующую ошибку, например toast или короткое сообщение рядом с графиком.
5. Если API вернул пустой массив, показать состояние «нет данных за выбранный период».

#### Очистка лишних `useEffect`

1. Проверить код после третьей домашки и убрать эффекты, которые не нужны.
2. Не использовать `useEffect` для данных, которые можно получить синхронно из текущего state/props:
   - расчет результата конвертации;
   - поиск выбранной валюты по коду;
   - формирование заголовков и описаний;
   - вычисление активной пары.
3. Оставлять `useEffect` только для настоящих side effects:
   - запросы к API;
   - подписки/таймеры;
   - синхронизация с внешними системами.
4. В коде или в `README` коротко объяснить принятые решения:
   - какие `useEffect` остались и почему;
   - какие `useEffect` были удалены и чем заменены;
   - почему выбран именно такой dependency array;
   - как работает cleanup.

### Источник данных

Данные нужно получать из локального backend-приложения в папке `server`.

Как запустить backend:

1. Убедиться, что установлен `.NET 8 SDK`.
2. Открыть терминал в папке `server`.
3. Выполнить `dotnet run`.
4. После запуска backend будет доступен по адресу `http://localhost:5081`, Swagger - по адресу `http://localhost:5081/swagger`.

Эндпоинт для графика:

```http
GET /prices?paymentCurrency=PLN&purchasedCurrency=CAD&fromDateTime=2026-05-29T10:00:00.000Z&toDateTime=2026-05-29T10:05:00.000Z
```

Пример ответа:

```json
[
  {
    "purchasedCurrencyCode": "CAD",
    "paymentCurrencyCode": "PLN",
    "price": 0.945,
    "dateTime": "2026-05-29T10:03:09.000Z"
  }
]
```

### UI

Ориентир по виду графика: ![chart](../chart-no-filter.jpg)

Требования к UI:

1. График должен быть встроен в текущий экран конвертера, без отдельной страницы.
2. Выбранная валютная пара на графике должна совпадать с валютной парой в селектах.
3. При `Swap` график должен обновляться для новой пары.
4. Можно использовать готовую библиотеку для графика или реализовать простой график самостоятельно.
5. Для этой домашки рекомендуется использовать `Recharts`: https://recharts.github.io/ . Это самый простой способ быстро собрать line chart в React без лишней низкоуровневой настройки.

Для тестов таймера используйте fake timers из `Vitest`.

### Ограничения

- Не смешивать DTO и клиентские модели: данные из API должны проходить через маппер, как в третьей домашке.
- Не плодить `useEffect` для синхронных вычислений.
- Не оставлять таймеры и незавершенные запросы без cleanup.
- Приложение не должно падать, если backend временно недоступен.

### Критерии приемки

- График показывает историю курса для выбранной валютной пары.
- График обновляется через `useEffect` при смене пары, смене периода и автоматически по таймеру.
- Актуальный курс и результат конвертации берутся из последней точки графика.
- Реализованы loading/error/empty-состояния графика.
- Старые запросы не могут перезаписать новые данные.
- Таймеры и запросы корректно очищаются.
- Лишние `useEffect` удалены или обоснованы.
- В `README` есть короткое объяснение решений по `useEffect`.
- Проект запускается через `npm run dev`.
- Проект корректно билдится через `npm run build`.
- Тесты проходят через `npm run test`.
- Нет ошибок при проходе линтерами
